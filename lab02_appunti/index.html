Il file è l'unità di memorizzazione su memoria di massa (archivio di dati).

La directory, o cartella (folder), è un particolare file utilizzato per raggruppare e organizzare i file in una struttura gerarchica.

> Nota operativa per Windows che di default nasconde le estensioni dei dipi di file conosciuti. Lanciare Esplora file (WINDOWS+E) e selezione la casella "Visualizza" (riferimento Virus I Love You).

È importante dare dei nomi significativi alle risorse utilizzate (file e directory).
  1. contenuto (Che cosa contiene/comunica?) TXT
  2. struttura (Che valore/senso/significato hanno le parti del documento) HTML
  3. presentazione/stile (Come viene fruito (es. visualizzato) dall'utente il contenuto?) CSS
  4. comportamento (Come si modifica/reagisce il documento quando l'utente interagisce con esso?) JavaScript

Un protocollo è un insieme di regole condivise per l'interazione/trasmissione/comunicazione.

L'XML è un metalinguaggio o tecnologia per la definizione di linguaggi di marcatura.

Possiamo classificare i documenti in base al dominio (o ambito) informativo a cui appartengono.

Per esempio, gli ipertesti, le immagini vettoriali 2D, ... 3D, le formule chimiche, le formule matematiche, le fatture elettroniche, gli spartiti musicali,

Un documento XML è un documento di testo arricchito di marcatori posti tra parentesi angolari (<>). Un elemento XML ha il seguente formato: nometag>contenuto</nometag>

Un documento XML si dice well formed (o ben formato) se rispetta le seguenti regole:

  1. ha il prologo <?xml version="1.0"?> (in Windows per inserire il carattere backtick, o apostrofo rovesciato, ALT+96, il numero va inserito col tastierino numerico)
  2. esiste uno e un solo tag radice
  3. ogni tag aperto va chiuso, per es. <a>...</a> (se l'elemento è vuoto si può utilizzare la forma compatta <a \>) . Es. <br /> è un tag vuoto che indica un'interruzione nel contenuto, break rule
  4. i tag vanno opportunamente annidati (o nidificati), ovvero l'ultimo tag aperto è il primo a essere chiuso, per es. <a>...<b>...</b>...</a>
  5. il linguaggio è case sensitive (es. <a> è diverso da <A>)
  6. gli attributi dei tag vanno delimitati con apici singoli o doppi. Es. di attributo <img src="immagine.png" />Nota: per inserire i simboli di minore < e maggiore > può essere necessario utilizzare le seguenti entità XML &lt; (less than) e &gt; (greater than)

Marcatura del testo della poesia Mattina di Giuseppe Ungaretti.

  1. Salva il testo della poesia nel file mattina.txt
Mattina
M'illumino d'immenso
1917, Giuseppe Ungaretti
  1. crea una copia del file rinominandola mattina1.xml
  2. marca le parti del testo con tag opportuni.
&lt;?xml version="1.0"?&gt;
&lt;titolo&gt;Mattina&lt;/titolo&gt;
&lt;testo&gt;M'illumino d'immenso&lt;/testo&gt;
&lt;anno&gt;1917&lt;/anno&gt;, &lt;autore&gt;Giuseppe Ungaretti&lt;/autore&gt;
  1. Crea una copia del file dell'esercizio precedente salvandola come mattina2.xml
  2. aggiungi la direttiva XML per associare un foglio di stile (stylesheet) <?xml-stylesheet type = "text/css" href = "styles.css"?>
  3. crea il file styles.css vuoto.
  4. visualizza il documento XML in Chrome/Edge/Firefox. Osserva che il contenuto è su un'unica riga (quindi gli a capo non sono rispettati) e non si vedono i tag.
  5. aggiungi il seguente codice CSS al file styles.css:
titolo, testo {
    display: block;
}
testo {
    font-style: italic;
}
titolo {
    font-weight: bold;
}
autore {
    color: blue;
}
  1. ricarica il documento XML in Chrome/Edge/Firefox. Osserva che il contenuto è formattato secondo le regole di stile precedenti.

Un linguaggio/grammatica/applicazione XML definisce per uno specifico dominio informativo:

  • un vocabolario
  • una sintassi specifica, ovvero come i tag e gli attributi possono essere utilizzati. Per es. nel documento precedente il tag <poesia> è il tag radice.
  • una semantica

Un documento XML si dice valido rispetto a una grammatica se:

  1. è well formed
  2. rispetta le regole specifiche (sintassi/semantica) del linguaggio
